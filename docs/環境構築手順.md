# 環境構築手順 — Blood Cell AI TensorFlow版

## 前提
- **OS**: Windows 11
- **GPU**: AMD Radeon RX 7900 XT (DirectML使用。CUDA/ROCm不要)
- **プロジェクトパス**: `C:\Users\hello\Desktop\project\bld-cell-ai-TF`

## 1. Minicondaインストール
公式サイトからWindows用インストーラーをダウンロード＆実行:
- URL: https://repo.anaconda.com/miniconda/Miniconda3-latest-Windows-x86_64.exe
- インストール先: `C:\Users\hello\miniconda3`（デフォルト）
- 「Add Miniconda3 to my PATH environment variable」にチェックを入れる

確認:
```powershell
conda --version
```

## 2. conda環境作成
```powershell
cd C:\Users\hello\Desktop\project\bld-cell-ai-TF
conda create -p .conda\env python=3.10 pip -y
```
※ Python 3.10必須（DirectML Pluginがcp310のみ対応）

## 3. パッケージインストール（順序重要）
```powershell
# ① TF本体を明示的にインストール（公式推奨手順）
.\.conda\env\Scripts\pip.exe install tensorflow-cpu==2.10

# ② DirectML Plugin（①の後にインストール）
.\.conda\env\Scripts\pip.exe install tensorflow-directml-plugin

# ③ NumPy 1.xに固定（TF 2.10はNumPy 2.xと非互換）
.\.conda\env\Scripts\pip.exe install "numpy<2"

# ④ 追加パッケージ
.\.conda\env\Scripts\pip.exe install scikit-learn matplotlib japanize-matplotlib seaborn pillow tqdm pyyaml jupyter medmnist
```
⚠️ `tensorflow`（メタパッケージ）や`tensorflow-gpu`は絶対に入れないこと。依存関係が壊れる。

## 4. 動作確認
```powershell
.\.conda\env\python.exe -c "import tensorflow as tf; print('TF:', tf.__version__); print('GPU:', tf.config.list_physical_devices('GPU'))"
```
期待される出力:
```
Successfully opened dynamic library ...directml.dll
DirectML: creating device on adapter 0 (AMD Radeon RX 7900 XT)
TF: 2.10.0
GPU: [PhysicalDevice(name='/physical_device:GPU:0', device_type='GPU')]
```

## 💡 補足：既に動いている環境の場合
`tensorflow-directml-plugin`を先に入れた環境でも`tensorflow-cpu 2.10.0`は自動導入されるため、GPU認識が確認できていれば入れ直しは不要。上記手順は**新規クリーンインストール時の推奨順序**。

## ⚠️ トラブルシューティング
| 問題 | 原因 | 対処 |
|---|---|---|
| `AttributeError: _ARRAY_API not found` | NumPy 2.x | `pip install "numpy<2"` |
| TF 2.15や2.21が入る | `tensorflow`メタパッケージが入った | `pip uninstall tensorflow`して`tensorflow-cpu==2.10`を入れる |
| GPUが認識されない | DirectML DLL未ロード | 環境変数設定（下記参照） |
| `tensorflow-hub`でTF 2.21が入る | 依存競合 | `tensorflow-hub`は未インストール。別途確認が必要 |
| パス長260文字超過 | Windows制限 | プロジェクト名を短縮するか短いパスに環境を作成 |
| `tf.test.is_built_with_gpu_support()`が`False` | DirectMLはCUDA非使用のため | 正常動作。`list_physical_devices`で確認すること |

### GPU認識トラブル時の環境変数
```powershell
# DirectML DLLのパスを明示（DLLが見つからない場合）
$env:TF_DIRECTML_PATH = "C:\Users\hello\Desktop\project\bld-cell-ai-TF\.conda\env\Lib\site-packages\tensorflow-plugins\directml"

# 詳細ログを有効化（デバッグ用）
$env:TF_CPP_MIN_LOG_LEVEL = "0"
```

## ⚠️ 注意事項
- **TF 2.15は使用不可**: DirectML PluginはTF 2.10.x用。2.15とは互換性なし
- **`tensorflow-directml`は非推奨**: TF 1.x用のため使用不可。`tensorflow-directml-plugin`を使用
- **DirectML Pluginは開発停止**: Microsoft公式でdiscontinued。教育・ポートフォリオ目的なら十分使用可能だが、本格的なパフォーマンスを求める場合はPyTorch + DirectMLまたはONNX Runtimeへの移行も検討
